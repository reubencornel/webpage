This is book is a highly recommended book if you are going to be taking programming interviews.

2 Approaches to programming problems

  • These problems tend to be hard so don't fret.
  • The interview questions tend to get harder and harder.
  • Be comfortable with the syntax of the language you are going to use in the interview.
  • The point of these interviews is to gauge the thought process. So perhaps its a good idea to keep talking to the inteviewer about what you are thinking

2.1 Solving the problem

  • Understand the problem completely: This generally happens to be more hard than said here. Ask clarifying questions if required.
  • Try out an example once you have understood the problem
  • Focus on getting the algorithm right
  • While coding explain what you are doing
  • Verify if the code is correct by tracing through the code
  • Check all error cases and boundary conditions
  • If you are writing too much of code, good probability you are doing something wrong.
  • Analyze the solution

Why do we need link lists

First we need to see why are link lists needed in the first place. We could comfortably use arrays and get on with life. But the problem with arrays is that it allocates memory statically. As a result when we require to use programs where we don't know that amount of data that is going to be input we can not use arrays. As an example take a look at the program below. You can copy and paste the program below in an editor and compile it, and run it.


0     #include<stdio.h>
1     int main(){
2          /*variable declaration*/
3          int i=0;
4          int list[5];
5          int entry_number=0;
6          int limit=50;
7         
8          /*fill up list up to the brim*/
9          for(i=0;i<limit;i++){
10              list[i]=entry_number;
11              entry_number++;
12         }
13         
14          /*try printing the list*/
15          for(i=0;i<limit;i++){
16              printf("%d\n",list[i]);
17          }
18     }


If you are using gnu/Linux you will see that after displaying 50 numbers the program puts out a segmentation fault message. This message occurs when ever a program tries to use a part of the memory that has not been allocated to it. As in this case we fill up 50 values in an array of size 10 and try accessing the part we have not allocated. Although this may work, as in not give a segmentation fault, its behaviour would be unpredictable. Its probably not a good idea to use it because the unallocated memory could be written over with out warning and the program could behave unexpectedly.
